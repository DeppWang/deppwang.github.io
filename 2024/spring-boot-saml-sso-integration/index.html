Spring Boot 集成 SAML 单点登录

发表于 2024-11-01 | 更新于 2024-11-01 | | 阅读次数

在现代应用程序中，单点登录（SSO）是一种流行的身份验证方式。通过 SSO，用户只需登录一次，即可访问多个系统和应用程序。本文将介绍如何在 Spring Boot 项目中集成 SAML SSO，使用 Authing 作为身份提供者（IdP）。

什么是单点登录

单点登录（SSO）是一种身份验证过程，允许用户使用单一的登录凭据访问多个应用程序。它简化了用户体验，减少了密码管理的复杂性。

什么是 SAML

SAML（Security Assertion Markup Language）是一种用于实现 SSO 的标准化协议。它通过在服务提供者（SP）和身份提供者（IdP）之间传递安全断言来实现身份验证。

区分前后台请求问题

在集成 SAML SSO 时，可能会遇到前后台请求区分的问题。由于前台和后台应用共享同一个 SSO 登录，可能会导致请求混淆。可以通过以下方法解决：

  • 使用 Session 或 Cookie 区分请求，但需注意多节点部署和安全性问题。
  • 使用 relay_state 在 SAML Request 中设置区分标识。

常见问题及解决方案

  1. idp-metadata 文件格式化后无法验证

    • 确保文件格式正确，使用在线工具进行验证。

  2. SAML 登录后需要额外短信验证

    • 可在 SAML Response 中添加自定义验证逻辑。

  3. Response issue time 错误

    • 确保服务器时间同步，开启自动校验时间。

  4. Response doesn’t have any valid assertion 错误

    • 检查 Authing 配置，删除 cookie 后重新登录。
